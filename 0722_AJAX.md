## 1. 객체 (Object)

**키로 구분된 데이터 집합(data collection)을 저장하는 자료형**

### 1-1. 구조 및 속성

- 중괄호 `{}`로 작성, 내부에는 `key: value` 쌍으로 구성된 **속성(property)** 을 여러 개 작성
- key는 문자형만 허용, value는 모든 자료형 허용

```javascript
const user = {
  name: 'Alice',
  'key with space': true,
  greeting: function () {
    return 'hello'
  },
}
```

**속성 참조**
- 점(`.`, chaining operator) 또는 대괄호(`[]`)로 접근
- key 이름에 띄어쓰기 같은 구분자가 있으면 **대괄호 접근만 가능**

```javascript
console.log(user.name)                 // Alice (조회)
console.log(user['key with space'])    // true
user.address = 'korea'                 // 추가
user.name = 'Bella'                    // 수정
delete user.name                       // 삭제
console.log('greeting' in user)        // true — 'in' 연산자: 속성 존재 여부 확인
```

### 1-2. 객체와 함수

**Method**
- 객체 속성에 정의된 함수. `object.method()` 방식으로 호출
- 메서드는 객체를 '행동'할 수 있게 함 → `'this'` 키워드를 사용해 객체에 대한 특정한 작업 수행 가능

**'this' keyword**
- 함수나 메서드를 **호출한 객체**를 가리키는 키워드
- 함수 내에서 객체의 속성 및 메서드에 접근하기 위해 사용

```javascript
const person = {
  name: 'Alice',
  greeting: function () {
    return `Hello my name is ${this.name}`
  },
}
console.log(person.greeting())  // Hello my name is Alice
```

- JavaScript에서 this는 함수를 **"호출하는 방법"에 따라** 가리키는 대상이 다름

| 호출 방법 | 대상 |
|---|---|
| 단순 호출 `myFunc()` | 전역 객체 (window) |
| 메서드 호출 `obj.myFunc()` | 메서드를 호출한 객체 |

**중첩된 함수에서의 this 문제점과 해결책**

```javascript
// 문제: forEach의 인자로 작성된 콜백은 "일반적인 함수 호출"이라 this가 전역 객체를 가리킴
const myObj2 = {
  numbers: [1, 2, 3],
  myFunc: function () {
    this.numbers.forEach(function (number) {
      console.log(this)  // window
    })
  },
}

// 해결: 화살표 함수는 자신만의 this를 가지지 않기 때문에
//       외부 함수(myFunc)에서의 this 값을 가져옴
const myObj3 = {
  numbers: [1, 2, 3],
  myFunc: function () {
    this.numbers.forEach((number) => {
      console.log(this)  // myObj3
    })
  },
}
```

**this 정리**
- JavaScript의 함수는 호출될 때 this를 암묵적으로 전달받으며, this는 함수가 "호출되는 방식"에 따라 결정되는 현재 객체를 나타냄
- Python의 self와 Java의 this가 선언 시 이미 값이 정해지는 것에 비해, **JS의 this는 호출되기 전까지 값이 할당되지 않고 호출 시에 결정됨 (동적 할당)**
- 장점: 함수(메서드)를 하나만 만들어 여러 객체에서 재사용 가능
- 단점: 이런 유연함이 실수로 이어질 수 있음 → 동작 방식을 충분히 이해하고 사용

### 1-3. 추가 객체 문법

**① 단축 속성** — 키 이름과 값으로 쓰이는 변수 이름이 같으면 단축 가능
```javascript
const name = 'Alice'
const age = 30
const user = { name, age }   // { name: name, age: age }와 동일
```

**② 단축 메서드** — 메서드 선언 시 `function` 키워드 생략 가능
```javascript
const myObj = {
  myFunc() { return 'Hello' },
}
```

**③ 계산된 속성 (computed property name)** — 키가 대괄호로 둘러싸인 속성. 고정 값이 아닌 변수 값 사용 가능
```javascript
const product = prompt('물건 이름을 입력해주세요')
const prefix = 'my'
const suffix = 'property'

const bag = {
  [product]: 5,
  [prefix + suffix]: 'value',
}
console.log(bag)  // {연필: 5, myproperty: 'value'}
```

**④ 구조 분해 할당 (destructuring assignment)** — 배열/객체를 분해하여 속성을 변수에 쉽게 할당
```javascript
const userInfo = {
  firstName: 'Alice',
  userId: 'alice123',
  email: 'alice123@gmail.com',
}

const { firstName } = userInfo
const { firstName, userId, email } = userInfo

// 활용: '함수의 매개변수'로 객체 구조 분해 할당 가능
function printInfo({ name, age, city }) {
  console.log(`이름: ${name}, 나이: ${age}, 도시: ${city}`)
}
printInfo(person)  // 호출 시 객체를 구조 분해하여 매개변수로 전달
```

**⑤ Object with '전개 구문'** — 객체 내부에서 객체 전개 = "객체 복사", 얕은 복사에 활용
```javascript
const obj = { b: 2, c: 3, d: 4 }
const newObj = { a: 1, ...obj, e: 5 }
console.log(newObj)  // {a: 1, b: 2, c: 3, d: 4, e: 5}
```

**⑥ 유용한 객체 메서드**
```javascript
const profile = { name: 'Alice', age: 30 }
Object.keys(profile)    // ['name', 'age']
Object.values(profile)  // ['Alice', 30]
```

### 1-4. [참고] JSON
- "JavaScript Object Notation" — key-value 형태로 이루어진 자료 표기법
- JS의 Object와 유사한 구조를 가지고 있지만 JSON은 형식이 있는 **"문자열"**
- JS에서 JSON을 사용하기 위해서는 **Object 자료형으로 변경해야 함**
- (보충) 변환 메서드: `JSON.parse(문자열 → 객체)`, `JSON.stringify(객체 → 문자열)`

## 2. 배열 (Array)

**순서가 있는 데이터 집합을 저장하는 자료구조** (Object는 순서 보장 X → 순서 있는 collection 필요)

### 2-1. 배열 구조
- 대괄호 `[]`를 이용해 작성, 요소 자료형 제약 없음
- `length` 속성으로 요소 개수 확인

```javascript
const names = ['Alice', 'Bella', 'Cathy']
console.log(names[0])       // Alice
console.log(names.length)   // 3
```

### 2-2. 주요 메서드

| 메서드 | 역할 |
|---|---|
| `push` / `pop` | 배열 **끝** 요소를 추가 / 제거 |
| `unshift` / `shift` | 배열 **앞** 요소를 추가 / 제거 |

### 2-3. Array Helper Methods
- 배열 조작을 보다 쉽게 수행할 수 있는 특별한 메서드 모음 (**ES6 도입**)
- 배열의 각 요소를 **순회**하며 각 요소에 대해 함수(**콜백함수**)를 호출 — `forEach()`, `map()`, `filter()`, `every()`, `some()`, `reduce()` 등
- **메서드 호출 시 인자로 함수(콜백함수)를 받는 것이 특징**

**forEach** — 배열의 각 요소를 반복하며 모든 요소에 대해 함수를 호출

```javascript
arr.forEach(callback(item[, index[, array]]))
```
- 콜백함수 매개변수 3가지: ① `item` 처리할 배열의 요소 ② `index` 요소의 인덱스(선택) ③ `array` forEach를 호출한 배열(선택)
- 반환 값: **undefined**

```javascript
const names = ['Alice', 'Bella', 'Cathy']

names.forEach(function (name) { console.log(name) })  // 일반 함수 표기
names.forEach((name) => { console.log(name) })        // 화살표 함수 표기

names.forEach(function (name, index, array) {
  console.log(`${name} / ${index} / ${array}`)
})
// Alice / 0 / Alice,Bella,Cathy ...
```

**map** — 배열의 모든 요소에 대해 함수를 호출하고, 반환된 호출 결과 값을 모아 **새로운 배열을 반환**

```javascript
arr.map(callback(item[, index[, array]]))
```
- forEach의 매개변수와 동일. forEach와 동작 원리는 같지만 **새로운 배열을 반환**한다는 차이

```javascript
// 배열을 순회하며 각 객체의 name 속성 값 추출 (for...of 와 비교)
const persons = [{ name: 'Alice', age: 20 }, { name: 'Bella', age: 21 }]

// for...of 방식
let result1 = []
for (const person of persons) { result1.push(person.name) }

// map 방식
const result2 = persons.map(function (person) { return person.name })
// ['Alice', 'Bella']

const numbers = [1, 2, 3]
const doubleNumber = numbers.map((number) => number * 2)  // [2, 4, 6]
```

### 2-4. 배열 순회 종합

| 방식 | 특징 | 비고 |
|---|---|---|
| `for loop` | 배열의 인덱스를 이용하여 각 요소에 접근, `break`·`continue` 사용 가능 | |
| `for...of` | 배열 요소에 바로 접근 가능, `break`·`continue` 사용 가능 | |
| `forEach` | 간결하고 가독성이 높음, callback 함수로 각 요소 조작 용이, `break`·`continue` 사용 불가능 | **사용 권장** |

### 2-5. 기타 Array Helper Methods (MDN 참고)

| 메서드 | 역할 |
|---|---|
| `filter` | 콜백 함수의 반환 값이 참인 요소들만 모아서 새로운 배열을 반환 |
| `find` | 콜백 함수의 반환 값이 참이면 해당 요소를 반환 |
| `some` | 요소 중 하나라도 콜백을 통과하면 `true` 반환 후 즉시 순회 중지, 모두 통과 못 하면 `false` |
| `every` | 모든 요소가 콜백을 통과하면 `true`, 하나라도 실패하면 즉시 `false` 반환 후 순회 중지 |

### 2-6. [참고] "배열은 객체다"
- 배열도 키와 속성들을 담고 있는 **참조 타입의 객체** — 배열의 키는 숫자(인덱스)
- 숫자형 키를 사용함으로써 객체 기본 기능 외에 "순서가 있는 컬렉션"을 제어하는 특별한 메서드를 제공하는 것
- 배열은 인덱스를 키로 가지며 `length` 속성을 갖는 특수한 객체 (`Object.getOwnPropertyDescriptors()`로 확인 가능)

## 3. 비동기 (Asynchronous)

### 3-1. 동기 vs 비동기
- **Synchronous (동기)**: 프로그램의 실행 흐름이 순차적으로 진행 — 하나의 작업이 완료된 후에 다음 작업이 실행
- **Asynchronous (비동기)**: 실행 흐름이 순차적이지 않으며, 작업이 완료되기를 **기다리지 않고** 다음 작업이 실행 → 작업의 완료 여부를 신경 쓰지 않고 **동시에 다른 작업들을 수행할 수 있음**
- 비동기 특징: **병렬적 수행**. 당장 처리할 수 없고 시간이 필요한 작업들은 별도로 요청을 보낸 뒤, 응답이 빨리 오는 작업부터 처리

### 3-2. Single Thread 언어, JavaScript
- Thread란? 작업을 처리할 때 **실제로 작업을 수행하는 주체** (multi-thread = 수행 주체가 여러 개)
- JavaScript는 한 번에 하나의 일만 수행하는 **Single Thread 언어** → 하나의 작업을 요청한 순서대로 처리할 수밖에 없음
- 그러면 어떻게 비동기 처리가 가능한가? → **JavaScript Runtime**
  - "JavaScript가 동작할 수 있는 환경(Runtime)" — 브라우저 또는 Node
  - JS 자체는 Single Thread이므로, 비동기 처리를 도와주는 환경이 필요

### 3-3. 브라우저 환경에서의 비동기 처리 요소 (★)

| 요소 | 설명 |
|---|---|
| **Call Stack** | 요청이 들어올 때마다 순차적으로 처리하는 Stack (**LIFO**). 기본적인 JS의 Single Thread 작업 처리 |
| **Web API** | JS 엔진이 아닌 **브라우저에서 제공**하는 runtime 환경. 시간이 소요되는 작업을 처리 (setTimeout, DOM Event, 비동기 요청 등) |
| **Task Queue** | 비동기 처리된 Callback 함수가 대기하는 Queue (**FIFO**) |
| **Event Loop** | 태스크가 들어오길 기다렸다가 들어오면 처리하고, 없으면 잠드는 끊임없이 돌아가는 JS 내 루프. Call Stack과 Task Queue를 지속적으로 모니터링 |

**동작 방식 (★ 순서 암기)**
1. 모든 작업은 **Call Stack(LIFO)** 으로 들어간 후 처리된다
2. 오래 걸리는 작업이 Call Stack으로 들어오면 **Web API**로 보내 별도로 처리하도록 한다
3. Web API에서 처리가 끝난 작업들은 곧바로 Call Stack으로 들어가지 못하고 **Task Queue(FIFO)** 에 순서대로 들어간다
4. **Event Loop**가 Call Stack이 비어 있는 것을 계속 체크하고, 비어 있다면 Task Queue에서 가장 오래된(가장 먼저 들어온) 작업을 Call Stack으로 보낸다

**런타임 시각화 예제 (강의 16단계 시각 자료의 코드)**
```javascript
console.log('Hi')

setTimeout(function myFunc() {
  console.log('Work')
}, 3000)

console.log('Bye')
// 출력: Hi → Bye → (3초 후) Work
```
- setTimeout은 Web API로 이동 → 3초 경과 후 콜백(myFunc)이 Task Queue로 → Call Stack이 비면 Event Loop가 push

**정리**: JS는 Single Thread 언어로 동기적 처리를 하지만, 브라우저 환경에서는 Web API에서 처리된 작업이 Task Queue를 거쳐 Event Loop에 의해 Call Stack에 들어와 순차적으로 실행됨으로써 **비동기 작업이 가능한 환경**이 됨

### 3-4. [참고] 비동기를 사용하는 이유 — "사용자 경험"
- 아주 큰 데이터를 불러온 뒤 실행되는 앱을 동기식으로 처리하면, 데이터를 모두 불러온 뒤에야 실행 로직이 수행되므로 사용자는 **앱이 멈춘 것 같은 경험**을 겪게 됨
- 즉, 비동기 처리는 사용자 경험(UX)을 위해 필수적

## 4. AJAX

### 4-1. 정의
- **Asynchronous JavaScript and XML**
- **XMLHttpRequest 기술을 사용해 복잡하고 동적인 웹 페이지를 구성하는 프로그래밍 방식**
- 비동기적인 웹 애플리케이션 개발을 위한 기술
- 페이지 전체가 다시 로드되지 않고 **HTML 페이지 일부 DOM만 업데이트** — 웹 페이지 일부가 다시 로드되는 동안에도 코드가 계속 실행되어 비동기적으로 동작
- 이름의 X는 XML이지만, 실제로는 모든 종류의 데이터를 다룰 수 있음 (요즘은 JSON 위주)

### 4-2. XMLHttpRequest(XHR) 객체
- 웹 브라우저와 서버 간의 비동기 통신을 가능하게 하는 JavaScript 객체
- 특징
  - 브라우저와 서버 간의 **네트워크 요청을 전송**할 수 있음
  - 사용자의 작업을 방해하지 않고 **페이지의 일부를 업데이트**할 수 있음
  - 이름에 XML이 들어가지만 XML뿐만 아니라 **모든 종류의 데이터**를 가져올 수 있음

```javascript
const xhr = new XMLHttpRequest()
xhr.open('GET', 'https://jsonplaceholder.typicode.com/posts')
xhr.send()

xhr.onload = function () {
  if (xhr.status === 200) {
    console.log(xhr.responseText)
  } else {
    console.error('Request failed')
  }
}
```

### 4-3. 기존 기술과의 차이

**기존 방식**
1. 클라이언트(브라우저)에서 form을 채우고 서버로 제출(submit)
2. 서버는 요청을 처리한 뒤 **새로운 웹 페이지 전체**를 작성해 응답
3. → 모든 요청마다 새로운 페이지를 응답받기 때문에 **계속 새로고침 발생**

**Ajax 방식**
1. 클라이언트에서 **XHR 객체 생성 및 요청**
2. 서버는 새로운 페이지를 만들지 않고 **필요한 부분의 데이터만** 처리 후 응답 (JSON 등)
3. → 필요한 데이터만 받아 기존 페이지의 **일부를 수정 (새로고침 X)**
4. 서버에서 모두 처리되던 데이터 처리의 일부가 클라이언트 쪽에서 처리되므로 **교환되는 데이터량과 서버 처리량이 감소**

### 4-4. 이벤트 핸들러와의 관계
- 이벤트 핸들러는 비동기 프로그래밍의 한 형태 (이벤트 발생 시 호출되는 콜백 함수 제공)
- HTTP 요청은 응답까지 시간이 걸릴 수 있는 **비동기 작업** → 이벤트 핸들러를 XHR 객체에 연결해 요청의 진행 상태 및 최종 완료에 대한 응답을 받음

## 5. Callback과 Promise

### 5-1. 비동기 처리의 단점
- 비동기 처리의 핵심: Web API로 들어오는 순서가 아니라 **작업이 완료되는 순서에 따라 처리** = 완료 순서를 보장하지 않음
- 개발자 입장에서 **코드의 실행 순서가 불명확** → 실행 결과를 예상하면서 코드를 작성할 수 없음

### 5-2. 비동기 콜백
- 비동기적으로 처리되는 작업이 **완료되었을 때 실행되는 함수**
- 연쇄적으로 발생하는 비동기 작업을 순차적으로 동작할 수 있게 함 → 작업의 순서·동작 제어, 결과 처리에 사용

```javascript
const asyncTask = function (callback) {
  setTimeout(function () {
    console.log('비동기 작업 완료')
    callback()  // 작업 완료 후 콜백 호출
  }, 2000)
}

asyncTask(function () {
  console.log('작업 완료 후 콜백 실행')
})
// 출력: (2초 후) 비동기 작업 완료 → 작업 완료 후 콜백 실행
```

### 5-3. 콜백 지옥 (Callback Hell)
- "결과를 받아서 다음 기능 수행" 패턴이 반복되면 콜백 안에 콜백이 계속 중첩됨
- 코드 형태가 피라미드 같다고 해서 **"Pyramid of doom(파멸의 피라미드)"** 라고도 부름

```javascript
fetchUserData(1, (err, user) => {
  if (err) { ... } else {
    fetchUserPosts(user.id, (err, posts) => {
      if (err) { ... } else {
        fetchPostComments(posts[0].id, (err, comments) => {
          ...
        })
      }
    })
  }
})
```

### 5-4. Promise
- **자바스크립트에서 비동기 작업을 처리하기 위한 객체**
- 비동기 작업의 **성공 또는 실패**와 관련된 결과나 값을 나타냄

```javascript
const fetchData = () => {
  return new Promise((resolve, reject) => {
    // 비동기 작업 수행 후
    // 성공 시 resolve(값), 실패 시 reject(에러) 호출
  })
}
```

**then과 catch**
- `then(callback)`: 요청한 작업이 **성공하면** callback 실행 — callback은 **이전 작업의 성공 결과를 인자로** 전달받음
- `catch(callback)`: then이 하나라도 **실패하면** callback 실행 — callback은 이전 작업의 실패 객체를 인자로 전달받음
- then과 catch는 모두 **Promise 객체를 반환** → 계속해서 **chaining** 가능 → 비동기 작업의 **순차적인 처리** 가능

```javascript
work1()
  .then((result1) => {
    // ...
    return result2
  })
  .then((result2) => {
    // ...
    return result3
  })
  .then((result3) => {
    console.log('최종 결과: ' + result3)
  })
  .catch((error) => {
    // 에러 처리
  })
```

### 5-5. Promise가 보장하는 것 (vs 비동기 콜백) ★
1. 콜백 함수는 JavaScript의 Event Loop가 **현재 실행 중인 Call Stack을 완료하기 이전에는 절대 호출되지 않음**
2. Promise의 callback 함수는 Event Queue에 배치되는 **엄격한 순서로 호출**됨
3. `.then()`을 여러 번 사용하여 **여러 개의 callback 함수를 추가**할 수 있음 (**Chaining**)

## 6. Axios

### 6-1. 정의
- 브라우저와 Node.js에서 사용할 수 있는, 웹 애플리케이션에서 **서버와 통신할 때 사용하는 JavaScript 라이브러리**
- 클라이언트-서버 사이의 HTTP 요청 생성과 응답 처리를 간편하게 해줌 (**Promise API 지원**)

**Ajax를 활용한 클라이언트-서버 간 동작 흐름**
1. 클라이언트: XHR 객체 생성 및 요청
2. 서버: 응답 데이터 생성
3. 서버 → 클라이언트: JSON 데이터 응답
4. 클라이언트: **Promise 객체 데이터를 활용해 DOM 조작** (웹 페이지의 일부분만 다시 로딩)

### 6-2. 사용법
```html
<!-- CDN 방식 -->
<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>
```
- 공식 문서: https://axios-http.com/

```javascript
// axios 객체를 활용해 요청을 보낸 후 응답 데이터 Promise 객체를 받음
axios({
  method: 'get',
  url: 'https://api.thecatapi.com/v1/images/search',
})
  .then((response) => {
    console.log(response)        // Response 객체
    console.log(response.data)   // 실제 데이터
  })
  .catch((error) => {
    console.log(error)
  })
```

### 6-3. 고양이 사진 가져오기 실습

**기본 — 응답 데이터 확인**
```html
<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>
<script>
  const URL = 'https://api.thecatapi.com/v1/images/search'
  axios({ method: 'get', url: URL })
    .then((response) => {
      console.log(response)
      console.log(response.data)
    })
    .catch((error) => {
      console.log(error)
      console.log('실패했다옹')
    })
  console.log('야옹야옹')
</script>
```
- 출력 순서: **'야옹야옹'이 먼저** 출력되고 그 뒤에 Response 도착 → axios 요청이 **비동기**이기 때문 (★)
- 응답 data는 `[{ id, url, width, height }]` 형태의 배열

**심화 — 버튼 클릭 시 이미지를 DOM에 추가 (3단계)**
1. 버튼을 작성하고 axios 동작을 콜백 함수로 작성, 이벤트 핸들러 부착
2. 응답 데이터에서 필요한 이미지 주소 값 찾기 → `response.data[0].url`
3. 찾은 이미지 주소를 활용해 `img` 태그 구성

```javascript
const URL = 'https://api.thecatapi.com/v1/images/search'
const btn = document.querySelector('button')

const getCats = function () {
  axios({ method: 'get', url: URL })
    .then((response) => {
      const imgUrl = response.data[0].url
      const imgElem = document.createElement('img')
      imgElem.setAttribute('src', imgUrl)
      document.body.appendChild(imgElem)
    })
    .catch((error) => {
      console.log(error)
      console.log('실패했다옹')
    })
}

btn.addEventListener('click', getCats)
```

### 6-4. 정리 — Ajax와 Axios (★)
- **Ajax**: 하나의 특정한 기술이 아니라, **비동기적인 웹 애플리케이션 개발에 사용하는 기술들을 묶어서 지칭** (개념)
- **Axios**: 클라이언트·서버 사이에 HTTP 요청을 만들고 응답을 처리하는 데 사용되는 JS **라이브러리** (도구, Promise API 지원)
- 앞으로: 프론트엔드에서 Axios를 활용해 **DRF로 만든 API 서버**로 요청을 보내고, 받아온 데이터를 처리하는 로직을 작성하게 됨

## 7. 추가로 알면 좋은 내용 (강의 범위 밖 보충)

### Promise의 3가지 상태
- `pending`(대기) → `fulfilled`(이행, resolve됨) 또는 `rejected`(거부, reject됨)
- 한번 확정(settled)되면 상태가 바뀌지 않음. 면접 단골 개념

### async/await
Promise의 문법적 설탕. 다음 단계(Vue + DRF 연동)에서 사실상 표준으로 쓰게 됨:
```javascript
const getCats = async function () {
  try {
    const response = await axios.get(URL)
    console.log(response.data[0].url)
  } catch (error) {
    console.log(error)
  }
}
```
- then 체인 대신 동기 코드처럼 읽힘. `await`는 async 함수 안에서만 사용 가능

### axios 단축 표기와 response 구조
- `axios({method: 'get', url})` 대신 실무에서는 `axios.get(url)`, `axios.post(url, data)` 형태를 주로 씀
- response 객체 구조: `{ data, status, statusText, headers, config }` → 실제 데이터는 항상 `response.data`

### fetch API와의 비교
- `fetch`는 브라우저 내장(설치 불필요)이지만 JSON 변환(`res.json()`)을 수동으로 해야 하고, 404/500 응답에도 reject되지 않음 → axios가 편한 이유

### Microtask Queue (Event Loop 심화)
- Promise의 then 콜백은 Task Queue가 아닌 **Microtask Queue**로 들어가며, Task Queue보다 **우선순위가 높음**
- 그래서 `setTimeout(..., 0)`보다 `Promise.resolve().then(...)`이 먼저 실행됨 — 출력 순서 문제로 자주 출제

```javascript
console.log('1')
setTimeout(() => console.log('2'), 0)
Promise.resolve().then(() => console.log('3'))
console.log('4')
// 출력: 1 → 4 → 3 → 2
```

### finally / Promise.all
- `.finally(callback)`: 성공/실패와 무관하게 항상 실행 (로딩 스피너 끄기 등)
- `Promise.all([p1, p2, ...])`: 여러 비동기 요청을 병렬 처리 후 결과를 한꺼번에 받음

### Optional chaining `?.` (객체 파트 보충)
- 이 강의 자료에는 없지만 객체 접근에서 자주 쓰는 문법: 참조 대상이 `null`/`undefined`면 에러 대신 `undefined` 반환
- `user.address?.street`, `user.method?.()` — API 응답처럼 값 존재가 불확실한 중첩 객체 접근에 유용

### CORS
- 프론트(Vue)에서 DRF 서버로 axios 요청을 보내면 거의 반드시 **CORS 에러**를 만남
- 서버 쪽에 `django-cors-headers` 설정이 필요하다는 것만 미리 기억해두면 관통 프로젝트에서 당황하지 않음
