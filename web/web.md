## HTML
> HyperText Markup Language : 웹 페이지의 의미와 *구조*를 정의하는 언어  
- HyperText : 웹 페이지를 다른 페이지로 연결하는 링크(참조를 통해 사용자가 한 문서에서 다른 문서로 즉시 접근할 수 있는 텍스트)
- Markup Language : 데이터의 구조를 명시하기 위한 언어 (글씨가 굵어보이고 커지는 건 CSS 개념) 

> HTML의 구조
- ```<!DOCTYPE html>``` : 문서가 html 문서라는 것을 명시
- ```<head></head>``` : 메타 데이터 포함
- ```<body></body>```에 보통 80%의 내용이 포함됨
- element(요소) : 여는 태그 & 닫는 태그 + 그 안의 내용으로 구성(닫는 태그가 없는 것도 있음 -> 본문이 없거나 본문을 담을 다른 방법이 있는 경우)
- attributes(속성) : 여러개의 p 태그를 하나의 그룹으로 나타내고 싶다? 그럼 class로 묶어 -> 이걸 속성이라고 함
  - 요소 이름과 속성 사이에 공백이 있어야 함
  - 속성 값은 열고 닫는 따옴표로 감싸야 함

> img 요소
- 상대경로 or URL으로 보여주기 가능

> Text Structure
- 단순히 텍스트를 크게만 만드는 것이 아니라 문서에 의미를 부여하는 것
- 구조화를 해놓으면 나중에 편하게 스타일링이 가능
- data crawling 시에도 구조를 알고있으면 수월하게 수집 가능

## CSS
> Cascading Style Sheet : 웹 페이지의 디자인과 레이아웃을 구성
- 구성 : 선택자 + 속성 & 값
- css 적용 방법 -> 인라인 안씀. 내부 스타일 시트도 권장하진 않음. 외부 스타일 시트를 권장한다!(파일 따로 만들고 불러오기)
