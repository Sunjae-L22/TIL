  ## CLI vs GUI
  - 왜 CLI를 쓸거냐. CLI가 무조건 GUI보다 빠름. 뭐든지!
  - 효율성, 정밀한 제어 강력한 자동화, 표준 환경(우린 이제 WSL, Linux, docker 사용할 예정)
  - 꼭 외워야 할 것들! 
    1. '.' : 현재 작업 위치
    2. '..' : 현재의 상위 디렉토리(부모 폴더) -> 1, 2번은 상대경로

  - 기초 문법
    1. touch : 파일 생성
    2. mkdir : 새 디렉토리 생성
    3. ls : 현재 작업 중인 디렉토리 (터미널 여는 법 : ctrl + `)
    4. mv (파일명) (경로) : (파일명)을 (경로)로 옯겨라. 명령어의 구분은 띄어쓰기로!(그래서 폴더명에 띄어쓰기 절대 금지)
    5. cd (경로) : (경로)로 작업중인 공간 옮겨가기.(change directory 약자)
    6. code (파일명) : (파일명) 파일 오픈
